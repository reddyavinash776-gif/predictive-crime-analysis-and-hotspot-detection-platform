# 🚨 Predictive Crime Analysis and Hotspot Detection (Crime Intelligence Command Center)

An interactive, machine-learning-powered spatial-temporal Crime Intelligence Command Center designed for law enforcement, municipal administrators, and urban planners to proactively analyze historical crime patterns, predict crime risk, and detect spatial hotspots.

---

## 🔹 Project Overview

This is a complete academic final-year project that utilizes Machine Learning, Geographic Information Systems (GIS), and Data Science to:
1. **Detect Hotspots:** Classify whether a specific neighborhood is a crime hotspot using a Random Forest Classifier.
2. **Forecast Future Risk:** Estimate the probability of crime occurrences (risk index) at specific hours and days of the week.
3. **Cluster Spatial Zones:** Partition coordinates using K-Means Clustering to visually group regions into High-Risk, Medium-Risk, and Low-Risk zones.
4. **Interactive GIS Visualization:** Render density heatmaps, spatial cluster maps, and risk gauges on a dark-themed Command Center Dashboard.

## 🔹 Dataset Schema

The system uses a highly realistic, NCRB-inspired Indian crime incident dataset (`dataset.csv`) comprising 6,000 incidents spanning 6 major Indian cities (**Hyderabad, Bangalore, Chennai, Mumbai, Delhi, Vijayawada**).

| Column Name | Type | Description |
|---|---|---|
| `crime_id` | String | Unique crime incident ID (e.g. `CRIME-1001`) |
| `date` | Date | Date of occurrence (YYYY-MM-DD) |
| `time` | Time | Time of occurrence (HH:MM:SS) |
| `city` | String | Name of the Indian City |
| `area` | String | Locality/Neighborhood (10 per city) |
| `latitude` | Float | GPS latitude of the incident |
| `longitude` | Float | GPS longitude of the incident |
| `crime_type` | String | Theft, Robbery, Burglary, Assault, Cyber Crime, Vehicle Theft, Fraud |
| `severity_level` | String | Low, Medium, High |
| `day_of_week` | String | Day of the week (e.g., Monday) |
| `month` | Integer | Month of occurrence (1 to 12) |
| `year` | Integer | Year of occurrence (2023 to 2025) |
| `arrest_made` | String | Yes / No status of suspect arrest |
| `crime_status` | String | Solved, Under Investigation, Unsolved |
| `hotspot_label` | Integer | Target variable (1 = Hotspot, 0 = Non-Hotspot) |

*Hotspot Definition:* An area is marked as a hotspot if its total crime volume is in the top 30% (70th percentile) of all areas within that specific city.

## 🔹 Machine Learning Models Details

### 1. Crime Hotspot Classification
* **Algorithm:** Random Forest Classifier
* **Objective:** Predict if a neighborhood will be a hotspot (`hotspot_label` 1/0) based on location and temporal attributes.
* **Accuracy:** Expected accuracy is ~90%+ due to the spatial aggregation patterns.

### 2. Future Crime Prediction (Risk Forecasting)
* **Algorithm:** Random Forest Classifier (Probabilistic Model)
* **Objective:** Estimate `Crime Risk Score` (0% to 100%) by predicting if a crime occurring in a given space-time window will be of Medium or High severity.
* **Output:** Displays risk gauges and future hazard predictions.

### 3. Crime Spatial Clustering
* **Algorithm:** K-Means Clustering ($K = 3$)
* **Objective:** Groups the GPS latitude and longitude coordinates into 3 geographical clusters. The clusters are ranked by incident count to automatically demarcate **High Risk**, **Medium Risk**, and **Low Risk** zones.

## 🔹 Viva Voce Preparation Guide (Students)

Be prepared to answer these common questions in your project defense:

1. **How is the Hotspot Label determined?**
   * *Answer:* It's dynamically determined using crime density. We group crime counts by `(city, area)` and calculate the 70th percentile of crime volume inside each city. Areas exceeding this threshold are flagged as hotspots (`1`), others are non-hotspots (`0`).
2. **Why K-Means for Clustering?**
   * *Answer:* K-Means is a centroid-based clustering algorithm. It is highly efficient for spatial grouping based on coordinates (Latitude/Longitude) to identify geographical clusters representing different risk zones.
3. **What features does the Random Forest use to predict risk?**
   * *Answer:* It utilizes `city_encoded`, `area_encoded`, `crime_type_encoded`, `hour`, `day_of_week_encoded`, and `month` to recognize spatial-temporal patterns (e.g. higher burglary rates during midnight hours in specific areas).
4. **How does Streamlit interface with your ML models?**
   * *Answer:* The trained models, label encoders, and coordinate scalers are saved as serialized files using `joblib`. The Streamlit app loads these models into memory on startup and runs live inferences when users interact with the sliders and dropdown menus.
